Sun Game Server Configuration Properties

This page documents the configuration properties supported by the Sun Game Server.

General Properties

These properties control various general facilities in the Sun Game Server.

com.sun.sgs.app.name
Specifies the application name.
com.sun.sgs.app.port
Specifies the TCP port to listen for client connections.
com.sun.sgs.txn.timeout 100
The maximum amount of time in milliseconds that a transaction will be permitted to run before it is a candidate for being aborted.
com.sun.sgs.txn.timeout.unbounded Long.MAX_VALUE
The maxmum amount of time in milliseconds that an unbounded transaction will be permitted to run before it is a candidate for being aborted.

DataService Properties

 These properties control the implementation of the DataService.

com.sun.sgs.impl.service.data.DataServiceImpl.detect.modifications true
Specifies whether to automatically detect modifications to managed objects. If set to something other than true, then applications need to call DataManager.markForUpdate or ManagedReference.getForUpdate for any modified objects to make sure that the modifications are recorded by the DataService.

DataStore Properties

 These properties are a subset of the properties supported by the DataService that control the implementation of the underlying DataStore.

com.sun.sgs.impl.service.data.store.DataStoreImpl.directory ${com.sun.sgs.app.root}/dsdb
The directory in which to store database files. Each instance of requires its own, unique directory.
com.sun.sgs.impl.service.data.store.DataStoreImpl.allocation.block.size 100
The number of object IDs to allocate at a time. This value must be greater than 0. Object IDs are allocated in an independent transaction, and are discarded if a transaction aborts, if a managed object is made reachable within the data store but is removed from the store before the transaction commits, or if the program exits before it uses the object IDs it has allocated. This number limits the maximum number of object IDs that would be discarded when the program exits.
com.sun.sgs.impl.service.data.store.DataStoreImpl.cache.size 1000000
The size in bytes of the Berkeley DB cache. This value must not be less than 20000.
com.sun.sgs.impl.service.data.store.DataStoreImpl.flush.to.disk false
Whether to flush changes to disk when a transaction commits. If false, the modifications made in some of the most recent transactions may be lost if the host crashes, although data integrity will be maintained. Flushing changes to disk avoids data loss but introduces a significant reduction in performance.

ClientSessionService Properties

 These properties control the implementation of the ClientSessionService.

com.sun.sgs.impl.service.ClientSessionServiceImpl.id.block.size 256
Specifies the block size to use when reserving session IDs. Must be greater than 8.

Kernel Properties

 These properties control the implementation of the Kernel.

com.sun.sgs.impl.kernel.Kernel.profile.level off
Whether to perform profiling, which is enabled if the value is on.
com.sun.sgs.impl.kernel.Kernel.profile.listeners
com.sun.sgs.impl.kernel.profile.AggregateProfileOpListener:
com.sun.sgs.impl.kernel.profile.SnapshotProfileOpListener:
com.sun.sgs.impl.kernel.profile.SnapshotParticipantListener
The set of ProfileOperationListeners that should be used. The value should be a colon-separated list of fully-qualified classes, each of which implements com.sun.sgs.kernel.ProfileOperationListener.
