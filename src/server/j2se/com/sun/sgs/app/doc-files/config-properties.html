Sun Game Server Configuration Properties

Services

General
    com.sun.sgs.app.name Specifies the application name.
    com.sun.sgs.txn.timeout 1000 The maximum amount of time in milliseconds that a transaction will be permitted to run before it is a candidate for being aborted.

ChannelService

com.sun.sgs.impl.service.channel.ChannelServiceImpl
    com.sun.sgs.app.name Specifies the application name.

DataService

 com.sun.sgs.impl.service.data.DataServiceImpl
    com.sun.sgs.impl.service.data.DataServiceImpl.debug.check.interval Integer.MAX_VALUE Specifies the number of DataService operations to skip between checks of the consistency of the managed references table. Note that the number of operations is measured separately for each transaction. This property is intended for use in debugging.
    com.sun.sgs.impl.service.data.DataServiceImpl.detect.modifications true Specifies whether to automatically detect modifications to managed objects. If set to something other than true, then applications need to call DataManager.markForUpdate or ManagedReference.getForUpdate for any modified objects to make sure that the modifications are recorded by the DataService.

DataStore

    com.sun.sgs.impl.service.data.store.DataStoreImpl.directory com.sun.sgs.app.root/dsdb
    The directory in which to store database files. Each instance of DataStoreImpl requires its own, unique directory.
    com.sun.sgs.impl.service.data.store.DataStoreImpl.allocation.block.size 100 The number of object IDs to allocate at a time. This value must be greater than 0. Object IDs are allocated in an independent transaction, and are discarded if a transaction aborts, if a managed object is made reachable within the data store but is removed from the store before the transaction commits, or if the program exits before it uses the object IDs it has allocated. This number limits the maximum number of object IDs that would be discarded when the program exits.
    com.sun.sgs.impl.service.data.store.DataStoreImpl.cache.size 1000000 The size in bytes of the Berkeley DB cache. This value must not be less than 20000.
    com.sun.sgs.impl.service.data.store.DataStoreImpl.flush.to.disk false Whether to flush changes to disk when a transaction commits. If false, the modifications made in some of the most recent transactions may be lost if the host crashes, although data integrity will be maintained. Flushing changes to disk avoids data loss but introduces a significant reduction in performance.
